## 參數比較實驗

以下五輪實驗皆固定由初始位置附近出發，目標點設為 `(5, 3, 2)`，只改變 `Kp` 或 `max_speed` 其中一個參數，以便比較其對飛行行為的影響。

| 設定 | 到達時間（約） | 觀察 |
| --- | --- | --- |
| `Kp = 0.5`, `max_speed = 1.0` | `10.6 s` | 作為 baseline。前段速度接近上限，接近目標後逐步減速；`y` 軸有輕微 overshoot，位置一度到 `3.35`，最後在 `(5.04, 3.19, 1.98)` 附近抵達。 |
| `Kp = 0.2`, `max_speed = 1.0` | `15.8 s` | 前段仍受速度上限影響，因此不會慢很多；但進入接近目標區域後，速度下降得較早，後半段收斂明顯較慢。幾乎沒有明顯 overshoot，整體較平順，最後在 `(4.82, 3.06, 1.93)` 附近抵達。 |
| `Kp = 1.0`, `max_speed = 1.0` | `9.0 s` | 前段同樣受到 `max_speed` 限制，因此和 baseline 差異有限；但接近目標時修正更積極，`x` 一度到 `5.25`、`y` 一度到 `3.35`，overshoot 較明顯，最後在 `(5.18, 2.95, 2.01)` 附近抵達。 |
| `Kp = 0.5`, `max_speed = 0.5` | `17.9 s` | 整段速度被壓低到約 baseline 的一半，移動較慢但較平順。仍有少量 `y` overshoot，但幅度不大，最後在 `(4.96, 3.19, 1.97)` 附近抵達。 |
| `Kp = 0.5`, `max_speed = 2.0` | `5.7 s` | 前段速度明顯提高，遠距離移動最快；但因 `Kp` 不變，接近目標時仍會自然降速，因此本次沒有出現明顯 overshoot，最後在 `(4.85, 3.05, 1.90)` 附近抵達。 |

## Appendix A 草稿：`Kp` 與 `max_speed` 的影響

在 `fly_straight.py` 中，控制器使用比例控制律 `v = Kp * error` 產生速度命令，再用 `max_speed` 對速度向量做上限限制。因此，`Kp` 主要影響誤差轉換成速度命令的強度，而 `max_speed` 則決定在距離目標較遠時，無人機最多可以飛多快。

由本次實驗可觀察到，降低 `Kp` 從 `0.5` 到 `0.2` 後，無人機接近目標時的速度下降得更早，整體反應較保守，抵達時間由約 `10.6 s` 增加到約 `15.8 s`，但 overshoot 也變小，飛行較平順。相反地，將 `Kp` 提高到 `1.0` 後，抵達時間縮短到約 `9.0 s`，接近目標時仍會下較大的修正命令，因此 `x` 與 `y` 軸都出現較明顯的 overshoot。這表示較大的 `Kp` 可以提升反應速度，但也可能讓系統在目標附近變得更積極、較不平順。

`max_speed` 的影響則主要體現在遠離目標時的移動速度。當 `max_speed` 由 `1.0` 降到 `0.5` 時，速度上限直接被壓低，抵達時間延長到約 `17.9 s`，但軌跡較溫和。當 `max_speed` 提高到 `2.0` 時，前段速度明顯提升，抵達時間縮短到約 `5.7 s`。不過本次實驗中，提高 `max_speed` 並沒有造成明顯 overshoot，原因是靠近目標後，`Kp * error` 本身已經變小，速度命令不再碰到上限，控制器仍會自然減速。換句話說，`max_speed` 主要影響長距離移動的收斂時間，而目標附近的細部行為仍較受 `Kp` 影響。
